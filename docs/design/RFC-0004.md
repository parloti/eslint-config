# RFC-0004: Consumer API Contract

**Status:** Draft

## Objective

Define the single supported consumer API for generating the package's flat ESLint configuration.

## Public Factory

The package exposes one configuration factory:

`config(options?: ConfigOptions): Promise<Linter.Config[]>`

Example usage:

```ts
import { config } from "@codeperfect/eslint-config";

export default config({
  disabledPlugins: ["jest", "jasmine"],
  rules: {
    "jsdoc/require-description": "warn",
  },
});
```

## Supported Options

### `disabledPlugins`

`disabledPlugins` disables one or more named modules from the composed configuration.

This is the only supported mechanism for removing package-defined behavior wholesale.

### `rules`

`rules` applies final rule overrides after all enabled modules have been composed.

Use this when a repository needs to tighten, loosen, or disable specific rules without forking a whole module.

### `boundaries`

`boundaries` provides repository-specific architecture input for the `boundaries` module.

Its structure is documented separately in RFC-0005 because it is repository topology, not generic API behavior.

## Default Behavior

When `config()` is called without options:

- all baseline modules are considered enabled
- modules that require unavailable optional integrations may resolve to `[]`
- `boundaries` does not activate unless valid repository input is supplied
- no consumer rule overrides are applied

## API Constraints

- Do not add alternate factories such as `createConfig()`, `all()`, or `strict()` unless they deliver new behavior that cannot be expressed through `config()`.
- Do not model module enablement as nested booleans per category. The named-module approach is more explicit and scales better as modules evolve.
- Do not leak third-party plugin configuration details into the public API unless the package is intentionally delegating ownership of that concern to consumers.

## Governance

CI may enforce that repositories use the package factory and may restrict certain local overrides, but those policy controls are external to the API itself.
